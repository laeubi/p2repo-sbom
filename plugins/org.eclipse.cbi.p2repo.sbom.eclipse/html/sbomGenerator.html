SBOM Generator

The SBOM (Software Bill of Materials) Generator creates a comprehensive inventory of all software components in your Eclipse installation in CycloneDX format.

Overview

A Software Bill of Materials (SBOM) is a formal, machine-readable inventory of software components and dependencies, information about those components, and their hierarchical relationships. This tool generates SBOMs for Eclipse installations in both XML and JSON formats following the CycloneDX specification.

Using the SBOM Generator

Accessing the Generator

To generate an SBOM for your Eclipse installation:

  1. From the menu bar, select Help > Generate SBOM...
  2. The SBOM Generator dialog will open with configuration options

Configuration Options

Installation Location

The Eclipse installation location is automatically detected and cannot be changed. This is the directory where Eclipse is installed and will be analyzed to generate the SBOM.

Output Files

  • XML Output: Specify the path where the XML format SBOM will be saved. The default location is in the parent directory of your Eclipse installation.
  • JSON Output: Specify the path where the JSON format SBOM will be saved. The default location is in the parent directory of your Eclipse installation.

You can leave either field empty if you don't want to generate that format.

Options

  • Verbose output: Enable detailed logging during the SBOM generation process. This is useful for troubleshooting or understanding what the generator is doing.
  • Search Maven Central: Query Maven Central repository for additional information about artifacts. This can provide more complete metadata but requires internet connectivity and may slow down the generation process.
  • Fetch advisories: Query the OSV (Open Source Vulnerabilities) database for security advisories related to the components. This requires internet connectivity.
  • Process Bundle-ClassPath: Analyze and include nested JAR files that are referenced in OSGi Bundle-ClassPath manifest entries. This provides a more complete inventory but may increase processing time.

Generating the SBOM

  1. Configure the desired output locations and options
  2. Click Generate to start the process
  3. The generation will run as a background job
  4. Progress can be monitored in the Progress view
  5. Upon completion, the SBOM files will be available at the specified locations

Understanding the SBOM

The generated SBOM contains:

  • Complete list of all Eclipse plugins and features
  • Version information for each component
  • License information where available
  • Component relationships and dependencies
  • Package URLs (PURLs) for component identification
  • Hash values for artifact verification
  • External references (websites, source repositories, issue trackers)
  • Security advisories (if fetch advisories option is enabled)

Viewing the SBOM

The generated SBOM files can be:

  • Viewed in any text editor
  • Processed by SBOM analysis tools
  • Imported into dependency tracking systems
  • Used for license compliance reporting
  • Analyzed for security vulnerabilities
